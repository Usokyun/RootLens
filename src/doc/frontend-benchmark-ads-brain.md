# RootLens 前端对标提炼：`ads_brain_fe`

> 标杆项目：`/Users/bytedance/my_project/ads_brain_fe`
>
> 目的：提炼其适合 RootLens 复用的前端设计方法，而不是照搬业务词汇、品牌元素或页面内容。

---

## 1. 为什么以它为标杆

`ads_brain_fe` 的长处不在单个组件，而在于它把复杂业务页面稳定地组织成了**工作台（workbench）**：

1. 有固定的壳层骨架，用户始终知道自己处在哪条业务链路里。
2. 主舞台、辅助面板、状态面板之间层级清晰，不会把所有控件挤在一个大卡片里。
3. 信息密度可以切换，复杂任务既能概览也能深挖。
4. 对移动端和窄屏有退化策略，浮层会自动转为堆叠。

这和 RootLens 的目标高度一致。RootLens 不是营销展示页，而是**多源工业 RCA 的探索性工作台**。

---

## 2. 提炼出的设计思路

### 2.1 先搭工作台骨架，再填业务内容

标杆项目先稳定四件事：

1. 顶部全局壳层：品牌、会话、全局偏好入口。
2. 左侧导航：始终固定在主链路页面之间切换。
3. 内容区滚动：滚动发生在内容区，不影响导航和全局定位。
4. 页面级 Hero：每页进入先给用户一张“你现在在做什么”的摘要卡。

RootLens 应复用这个顺序。先保证 `materials / evidence / graphs` 三页都是工作台，再谈具体 RCA 交互。

### 2.2 页面不是“一个大卡片”，而是“主舞台 + 辅助面板”

标杆项目很少把所有内容塞进单层列表，而是把页面拆成三类区域：

1. **Hero / Summary**：当前任务、关键统计、主动作。
2. **Board / Stage**：真正承载业务操作的主区域。
3. **Support Panels**：筛选、状态、偏好、详情、局部解释。

RootLens 的映射应该是：

1. `materials`：素材库是主舞台，导入和构图动作是辅助面板。
2. `evidence`：Observation 列表是主舞台，Case 上下文和当前 Observation 详情是辅助面板。
3. `graphs`：图谱画布是主舞台，RCA 状态、候选排序和悬浮信息是辅助面板。

### 2.3 控件要贴近任务阶段，而不是平均分布

标杆项目把控件按用户心智阶段摆放：

1. 先选上下文。
2. 再看结果。
3. 最后做调参或跳转。

RootLens 里也应保持这个顺序：

1. 先选图谱 / case / observation。
2. 再看 Route 1 / Route 2 / Graph highlight。
3. 最后做导入、重算、反馈或跳页。

### 2.4 通过“统一面板语法”降低认知切换

标杆项目的卡片、浮层、侧栏虽然用途不同，但语法一致：

1. 圆角卡片。
2. 浅层渐变背景。
3. 顶部轻量 header。
4. 数据用 pill/tag/metric card 表达。
5. Hover 只做轻微抬升，不做夸张动画。

RootLens 也应统一：

1. Section Card 用来承载大块内容。
2. Metric Card 用来显示数量、状态、阶段。
3. Floating/Sticky Panel 用来放详情、情报和偏好。

### 2.5 信息密度必须可切换

标杆项目大量使用布局切换和浮层收展：

1. 网格 / 列表切换。
2. 浮层展开 / 收起。
3. 大屏浮动，小屏堆叠。

这对 RootLens 很关键，因为同一份 Evidence/Graph 数据需要兼顾：

1. 快速扫览。
2. 对单条 observation 或单个 candidate 做精读。

### 2.6 动效只服务于结构感

标杆项目的动效集中在三类：

1. 页面初次进入时的轻量分段出现。
2. 卡片 hover 的轻微抬升。
3. 浮层打开/关闭的短时位移。

RootLens 应该避免“到处都动”，只保留对认知分层有帮助的动效。

### 2.7 浮层不是装饰，而是减少切页

标杆项目里浮动筛选面板、状态面板、偏好面板的价值是：

1. 让用户不离开主舞台就能看辅助信息。
2. 不把次级配置埋进深层弹窗。
3. 在窄屏自动落为普通块级区域。

RootLens 里最值得复用的不是“可拖拽”本身，而是“**辅助信息脱离主列表，但不脱离当前页面**”。

## 3. 可以直接复用的前端技巧

### 3.1 布局技巧

1. `sticky sider + scrollable content`：导航固定，内容区独立滚动。
2. `hero + board`：每页先摘要，再进入工作台。
3. `main board + sticky aside`：复杂任务页优先使用主区 + 粘性辅区。
4. `floating panel -> stacked panel`：宽屏悬浮，窄屏降级为普通卡片。

### 3.2 样式技巧

1. 用一组浅渐变和轻阴影建立层次，不用大面积纯白。
2. 用胶囊 tag 和 metric pill 表示结构化状态，不用长句堆说明文。
3. 卡片顶部保留轻量高亮线或色带，帮助识别模块语义。
4. 主按钮突出，次按钮退后，避免所有操作同权重。

### 3.3 交互技巧

1. 列表项 hover 有反馈，active 状态明显强于 hover。
2. 视图切换器做成实体开关，而不是普通按钮堆。
3. 输入筛选和状态提示放在离结果最近的位置。
4. 支持从当前卡片直接跳到下一工作台，缩短链路。

### 3.4 响应式技巧

1. 大屏保持左右分区，小屏改为纵向堆叠。
2. 浮层在窄屏不再绝对定位。
3. 列表模式优先服务小屏，网格模式优先服务大屏。

## 4. RootLens 的落地准则

### 4.1 不照搬品牌，只复用结构

RootLens 不应复制 `ads_brain_fe` 的品牌元素、OceanCloud 头图或业务词汇；只复用它的：

1. 工作台骨架。
2. 面板层次。
3. 信息密度切换。
4. 浮层/粘性辅助区策略。

### 4.2 不把 RCA 语义藏掉

RootLens 的特殊性是双路线 RCA，因此前端必须显式保留：

1. 当前 case / dataset 上下文。
2. Route 1 与 Route 2 的不同语义。
3. 图谱高亮与 evidence focus 的联动关系。
4. “本地重算 / 预生成 runtime / focus 同步”的状态说明。

### 4.3 三页改造目标

| 页面 | 对标后应达到的展示形态 |
|---|---|
| `materials` | Hero 摘要 + 素材主列表 + 粘性导入/构图侧栏 |
| `evidence` | Hero 摘要 + Case 侧栏 + Observation 主列表 + 当前 Observation 详情面板 |
| `graphs` | Hero 摘要 + 粘性 RCA 辅助列 + 图谱主舞台 + 浮动图谱情报卡 |

---

## 5. 本次改造的具体约束

1. 不改 RootLens 的核心数据 contract。
2. 不改变 `materials / evidence / graphs` 三页的主职责。
3. 优先做展示层与布局层重构，保留现有本地运行时逻辑。
4. 所有新增视觉模式都必须能在窄屏退化。

## 6. 实施清单

1. 新增统一的页面 Hero 组件，沉淀摘要卡语法。
2. 全局壳层改成“轻状态条 + 工作台内容区”。
3. `materials` 引入粘性导入侧栏与任务摘要。
4. `evidence` 补 Case 上下文/Observation 详情辅助面板。
5. `graphs` 强化图谱主舞台与 RCA 辅助列的分层。
6. 保持现有 list/grid 切换，并让其真正服务不同信息密度。

## 7. 本轮可直接复用的界面模式

### 7.1 紧凑页头，而不是重复大 Hero

当页面已经有全局壳层时，不要在内容区再次放一张大 Hero。更合适的是：

1. 保留统一的 `Current Workspace + 标题 + 描述` 紧凑页头。
2. 页面统计、阶段状态、数据集上下文放在右侧 tag 区。
3. 页面级动作如果必须保留，放在 tag 区下方的第二行，避免挤进正文卡片。

这套模式适合 RootLens 的三页：

1. `materials`：展示素材阶段、当前选中数、默认图谱。
2. `evidence`：展示 case / observation 总量，并把导入动作压到页头右侧。
3. `graphs`：展示当前数据集、case 数量和 RCA 状态。

### 7.2 当前焦点信息优先用模态框，不占永久侧栏

`ads_brain_fe` 的模态框值得复用的不是颜色，而是这组结构：

1. 外层 `panel` 使用顶部高亮线。
2. 左列给摘要、关键统计、主动作。
3. 右列给字段详情、tag、引用列表。
4. 详情信息只在用户需要时弹出，不长期压缩主列表宽度。

RootLens 的对应准则：

1. `Observation 当前焦点` 改为按钮触发的模态框。
2. 模态框内保留 `summary / detail / refs` 三层。
3. case 与 graph 上下文用 chips 展示，不再塞进页面固定侧栏。

### 7.3 导入区优先做“上拉展开 sheet”

对标项目里样本区的展开方式比传统虚线框更适合 RootLens：

1. 默认只显示明确动作按钮。
2. 需要拖拽时，再从按钮区上方向上展开导入 sheet。
3. sheet 自带收起动作，不长期占据工作台高度。

RootLens 中这条规则适用于素材管理：

1. 常用入口直接显示为三个按钮：上传文件、恢复历史会话、切换内置 Demo。
2. 拖拽导入区保持折叠，按需展开。
3. 选中素材预览和构图发布动作保持在同一辅助列中。

---

## 8. 对 RootLens 的最终判断

`ads_brain_fe` 最值得 RootLens 学的，不是“更像企业中后台”，而是下面这件事：

> **当页面承载的是持续分析任务，而不是一次性浏览任务时，界面必须先组织工作台，再组织内容。**

RootLens 的前端改造应以此为准。
