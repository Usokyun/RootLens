# RootLens UI 重设计文档

> 基于 2026-05-13 代码审计撰写，忽略现有前端实现，从数据模型与业务逻辑出发设计全新 UI。
>
> **设计原则**：三大页面覆盖完整工作流 —— 素材 → 证据 → 图谱分析。

---

## 一、数据与业务全景

### 1.1 核心实体关系

```
素材 (Source Files)
  │  图像 / 代码 / 语料 / 序列 (CSV/JSONL) / 节点边文件 ...
  │
  └── 构建 ──→ 知识图谱 (Graph Dataset)
                 │  nodes + edges, category/kind/relation/confidence
                 │
                 └── RCA ──→ 分析结果 (Analysis Result)
        ┌────────────────────┘          Route 1 + Route 2
        │
  证据 (Evidence)
    多模态异常观测 (variable / image_defect / log_event)
    entity hints → 链接到图谱节点
```

### 1.2 核心数据类型速览

| 实体 | 说明 |
|---|---|
| **素材** | 上传的原始文件（CSV/JSONL/JSON/图片等），用于构建图谱或提供 evidence |
| **Graph Dataset** | 一个知识图谱 = 一组 nodes + edges，有 id/label/graphKind |
| **Graph Node** | 节点：id, name, category, kind, aliases, degree, 原始 attributes |
| **Graph Edge** | 边：source→target, relation, confidence, weight, directed |
| **Case (Runtime Case)** | 一个分析案例 = 一个 Evidence + 指定的 Graph Dataset + AnalysisResult |
| **Observation** | 异常观测：variable / image_defect / log_event 三种 facet |
| **Route 1** | 实体链接 → 一致性检查 → 修正候选 → 排序路径 |
| **Route 2** | RBC 贡献度判定 → 候选枚举 → 传播仿真 → 根因排序 |
| **Cross-Route Signal** | 两条路线共同指向的候选 |
| **Analyst Feedback** | 用户对路径/候选的 accept/reject/note |
| **What-If Draft** | 修改 observation 后重新推理的草稿 case |

### 1.3 用户工作流

```
导入素材 ──→ 构建图谱 ──→ 导入证据 ──→ 在图谱上执行 RCA ──→ 反馈与假设
   │              │              │                │
   └── 素材管理 ──┘              │                │
                                 └── 证据管理 ────┘
                                                  └── 图谱探索 ──┘
```

## 二、页面总览

### 三个页面

> 侧边栏顺序（固定）：**证据管理 → 图谱探索 → 素材管理**

| # | 路由 | 名称 | 核心职责 |
|---|---|---|---|
| 1 | `/evidence` | **证据管理** | 导入多模态观测数据，生成统一 Evidence JSON，编辑与审查 |
| 2 | `/graphs` | **图谱探索** | 图谱可视化与分析中心：浏览、搜索、RCA、反馈、what-if、图谱编辑 |
| 3 | `/materials` | **素材管理** | 导入多模态素材，管理素材库，多选构建图谱 |

### 3.2 交互

**导入素材：**
- 点击上传 / 拖拽文件或文件夹
- 支持批量上传，自动识别文件类型（根据扩展名和内容）
- 上传后自动解析并在素材库中展示（显示文件名、类型图标、行数/尺寸、解析后的 layer/role）

**素材库：**
- 网格或列表视图展示所有已导入素材
- 每条素材卡片显示：文件类型图标 + 文件名 + 关键元信息（行数/尺寸/layer）
- 支持搜索（按文件名模糊匹配）和类型筛选
- **多选**：checkbox 选中多个素材 → 点击"构建图谱"
- 点击卡片可预览文件内容（CSV/JSONL 表格预览，图片放大查看，文本内容显示）

**构建图谱：**
- 选中素材后点击"构建图谱"按钮
- 系统自动检测素材组合：
  - `nodes.jsonl + edges.jsonl` → TEP 风格图谱
  - `nodes.csv + edges.csv` → MVTec 风格图谱
  - 其它组合 → 提示用户指定 node/edge 文件
- 构建完成后在"已构建的图谱"区域显示，同时自动添加到图谱探索页

**会话管理：**
- 支持导入 session bundle JSON 恢复完整会话
- 支持切换到内置 Demo 数据
- 底部显示当前 session 状态：模式 + 素材数 + 图谱数 + case 数 + 警告

### 4.2 交互

**Case 管理：**
- 左侧 Case 列表显示所有已导入/创建的 case
- 点击 case → 右侧显示该 case 的 observation 列表
- 新建 Case：手动创建空白 case，逐条添加 observation
- 导入 Evidence：上传 evidence JSON / CSV → 自动解析为 case
- 删除 case（带确认）

**Observation 列表（选中 case 后）：**
- 按 facet 用颜色标签区分：
  - `variable` — 蓝色，显示 variable_name + contribution + direction
  - `image_defect` — 橙色，显示 object + anomaly_type + severity
  - `log_event` — 绿色，显示 event_code + equipment
- 点击 observation → 右侧详情面板展示完整字段
- 支持按 facet 筛选、按 confidence/contribution 排序

**Observation 详情 + 编辑：**
- 完整字段展示，字段根据 facet 动态变化
- **编辑模式**：所有字段可编辑（表单控件根据字段类型适配）
- 编辑后保存 → 更新 evidence（标记为 draft）
- 支持重置到原始值

**实体链接预览：**
- 显示当前 observation 的 linked_entity_hints 映射到的图谱节点
- 展示匹配结果（top candidates + scores）
- 如果已执行过 RCA，显示 Route 1 的 linked_entities 结果

**Raw Evidence 引用：**
- 列表展示 raw_evidence_refs（文件路径 + 行号）
- 点击可定位到原始素材文件（如在素材库中存在）

**与图谱探索的联动：**
- "发送到图谱探索"按钮：将选中的 case 设为图谱探索页的当前分析目标，跳转到图谱探索页并自动触发 RCA

### 5.2 图谱导航（画布左上角）

- **图谱下拉选择框**：位于 KG canvas 左上角（原图谱名称位置）
  - 下拉项展示：dataset label、graphKind、节点数/边数
  - 选择后切换当前渲染图谱
  - 默认选中上一次查看的 dataset
- **RCA Case 选择**：
  - Case 选择保留在分析面板头部（来自证据管理页）
  - 选择后自动执行 RCA，结果展示在分析面板中
  - 也可不选 case，仅浏览图谱结构

### 5.3 图谱画布

**工具栏：**
- 返回按钮（从路径聚焦模式返回全局视图）
- 图谱选择下拉（位于画布左上角）
- 节点搜索输入框（搜索后居中定位）
- 筛选器下拉（按 category / kind / degree 范围过滤）
- 图例按钮（展开颜色映射）

**视图模式：**
- **全局视图**（默认）：展示完整力导向图
  - 拖拽节点、滚轮缩放
  - 点击节点 → 右侧详情面板更新
  - 双击节点 → 展开/折叠其邻居
  - 悬停边 → 浮层显示 relation + confidence
  - 节点根据 category 用不同颜色渲染
  - 边粗细根据 confidence/weight 调整
  - RCA 结果中的路径节点/边用高亮色标注
- **路径聚焦视图**：仅显示选定路径的节点和边，节点按路径顺序排列

**右键菜单（节点）：**
- 查看详情
- 展开邻居
- 隐藏此节点
- 在 Evidence 中查看关联 observation
- 编辑节点属性

**右键菜单（边）：**
- 查看详情
- 隐藏此边
- 编辑边属性

## 六、跨页联动

三个页面通过 AnalysisFocus 机制联动：

| 触发操作 | 来源 | 影响 |
|---|---|---|
| 素材管理中选中 case | Materials | Evidence 页展开该 case、Graphs 页选中对应图谱 |
| Evidence 中选中 observation | Evidence | Graphs 页高亮关联图谱节点 |
| Graphs 中选中 RCA 路径 | Graphs | Evidence 页高亮路径涉及的 observation |
| Graphs 中修改 observation (what-if) | Graphs | Evidence 页标记该 observation 有 draft |
| Materials 中新增图谱 | Materials | Graphs 页图谱列表自动刷新 |

实现方式：保留现有 `AnalysisFocusState` localStorage + CustomEvent 机制不变。

## 九、与原设计的差异

| 维度 | 原设计 (4 页) | 新设计 (3 页) |
|---|---|---|
| 页面结构 | `/import` `/graphs` `/evidence` `/reasoning` | `/evidence` `/graphs` `/materials` |
| 素材管理 | 混在 import 页面的导入流程中 | **独立页面**，素材库 + 多选构建 |
| Evidence | 纯浏览审查 | **管理 + 编辑**，支持新建/修改/删除 |
| RCA 分析 | 独立页面 `/reasoning` | **融入图谱探索页**作为分析面板 |
| 图谱编辑 | 不支持 | **支持**（添加/删除/修改节点和边） |
| 反馈管理 | 散落在 reasoning 页 | 集中在图谱探索页的反馈 Tab |

## 十、三个页面的核心价值

1. **素材管理**：用户的数据入口。不只是"导入"，而是 "管理素材库 → 多选构建图谱" 的完整流程。
2. **证据管理**：Evidence 的全生命周期管理。导入 → 生成 → 审查 → 编辑 → 修正，为 RCA 准备好高质量的证据。
3. **图谱探索**：核心分析中心。图谱可视化 + RCA 双路线分析 + 反馈决策 + What-If 假设 + 图谱编辑，所有分析活动都在这里闭环。
